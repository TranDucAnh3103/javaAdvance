1. GIỚI THIỆU
================================================================================

Dự án xây dựng hệ thống quản lý cửa hàng điện thoại dạng Console (CLI) 
sử dụng Java thuần + MySQL. Hệ thống phục vụ 2 loại người dùng:

  • ADMIN:  Quản lý danh mục, sản phẩm, đơn hàng, Flash Sale, Coupon, thống kê.
  • CUSTOMER: Duyệt sản phẩm, thêm giỏ hàng, áp mã giảm giá, đặt hàng.

Kiến trúc: 3-Layer Architecture (Presentation → Service → DAO)
Database: MySQL 8.x
Bảo mật: BCrypt hash password, PreparedStatement chống SQL Injection
Cấu hình: dotenv-java (.env) quản lý thông tin kết nối DB

  [✅] Quản lý phiên đăng nhập (Session) — lưu User hiện tại trong RAM
  [✅] Validate email (regex) và số điện thoại (10 số, bắt đầu bằng 0)
  [✅] Không cho phép bỏ trống trường bắt buộc (full_name, email, phone, password)

3.2 QUẢN LÝ DANH MỤC (Admin)
──────────────────────────────
  [✅] Hiển thị danh sách hãng sản xuất dạng bảng
  [✅] Thêm danh mục (kiểm tra trùng tên)
  [✅] Sửa danh mục (kiểm tra ID tồn tại)
  [✅] Xóa mềm danh mục (soft delete — đặt is_active = false)

3.3 QUẢN LÝ SẢN PHẨM (Admin)
──────────────────────────────
  [✅] Hiển thị danh sách SP đầy đủ thông tin (JOIN 3 bảng) + phân trang
  [✅] Thêm mới SP (validate giá > 0, stock >= 0)
  [✅] Sửa SP (không cho sửa ID, hiển thị dữ liệu cũ trước khi sửa)
  [✅] Xóa SP có xác nhận Y/N (soft delete)
  [✅] Tìm kiếm SP theo tên — tìm kiếm tương đối (LIKE %keyword%)
  [✅] Sắp xếp SP theo giá tăng/giảm (ASC/DESC)

3.4 MUA HÀNG (Customer)
─────────────────────────
  [✅] Hiển thị SP còn hàng (stock > 0), có phân trang
  [✅] Thêm SP vào giỏ hàng (kiểm tra tồn kho ảo — Virtual Stock Mapping)
  [✅] Quản lý giỏ hàng: xem, sửa số lượng, xóa sản phẩm
  [✅] Xác nhận đặt hàng → lưu Orders + OrderDetails (Transaction atomic)
  [✅] Tự động trừ tồn kho sau khi đặt hàng (WHERE stock >= qty chống Over-selling)
  [✅] Xem lịch sử đơn hàng và trạng thái

3.5 QUẢN LÝ ĐƠN HÀNG (Admin)
──────────────────────────────
  [✅] Hiển thị toàn bộ đơn hàng (JOIN users lấy tên khách hàng)
  [✅] Cập nhật trạng thái đơn hàng (State Machine 1 chiều):
       PENDING → SHIPPING → DELIVERED
       PENDING → CANCELLED
       SHIPPING → CANCELLED
       DELIVERED và CANCELLED = trạng thái cuối, không quay đầu

3.6 FLASH SALE — Giảm Giá Sản Phẩm (Admin + Customer)
───────────────────────────────────────────────────────
  [✅] Admin tạo sự kiện Flash Sale (tên, thời gian bắt đầu/kết thúc)
  [✅] Admin thêm SP vào Flash Sale (variant, % giảm, số lượng flash)
  [✅] Admin xem tất cả sự kiện (đánh dấu ĐAng CHẠY ★ / SẮP DIỄN RA / ĐÃ KẾT THÚC)
  [✅] Customer xem SP flash với tag [⚡] và giá đã giảm
  [✅] Customer mua hàng flash → giá flash ghi vào order_details
  [✅] Dual Stock Deduction: trừ đồng thời sale_stock + stock (cùng Transaction)

3.7 COUPON — Mã Giảm Giá Đơn Hàng (Admin + Customer)
──────────────────────────────────────────────────────
  [✅] Admin tạo coupon (code, % giảm, thời hạn, số lượt)
  [✅] Admin xem danh sách coupon (đánh dấu trạng thái: HẾT HẠN, HẾT LƯỢT, ĐANG DÙNG ★)
  [✅] Customer nhập mã giảm giá lúc checkout
  [✅] Validate 4 điều kiện: tồn tại? active? thời hạn? còn lượt?
  [✅] Tính tiền sau giảm: totalAmount × (100 - discount%) / 100
  [✅] Tự động trừ usage_limit sau checkout thành công

3.8 THỐNG KÊ (Admin)
─────────────────────
  [✅] Top N sản phẩm bán chạy nhất tháng (mặc định Top 5)
  [✅] Chọn tháng/năm để xem (mặc định tháng hiện tại)
  [✅] SQL aggregate: SUM(quantity), SUM(quantity × unit_price), GROUP BY product
  [✅] Loại đơn CANCELLED ra khỏi thống kê
  [✅] Hiển thị huy chương 🥇🥈🥉 cho Top 3

3.9 KIẾN TRÚC & KỸ THUẬT
──────────────────────────
  [✅] Kiến trúc 3 lớp: Presentation → Service → DAO
  [✅] Interface pattern cho Product, Order, FlashSale, Coupon, Statistics
  [✅] Custom Exception (6 loại) — tránh crash hệ thống
  [✅] PreparedStatement chống SQL Injection (toàn bộ DAO)
  [✅] Transaction JDBC (setAutoCommit(false)) cho Order, Flash Sale
  [✅] BigDecimal cho tính toán tiền tệ (tránh sai số float)
  [✅] BCrypt hash mật khẩu
  [✅] dotenv-java quản lý cấu hình DB

6.1 ĐĂNG NHẬP:
    User gõ email/pass → LoginRegister → UserLoginService.login() 
    → UserLoginDAO.getUserByEmail() → BCrypt.verify() → Session.setLoggedInUser()

    Customer xem SP (phân trang) → Nhập "b <ID>" → addToCart() (kiểm Flash price)
    → Xem giỏ → handleCheckout() → Nhập mã coupon (optional) → Chốt đơn [Y]
    → OrderService.checkout() → OrderDAOImpl.createOrder() [Transaction]

    → Nếu Flash Sale: deductSaleStock() (trừ sale_stock)
    → Nếu Coupon: deductUsage() (trừ lượt dùng)
    → COMMIT → Xóa giỏ hàng RAM
