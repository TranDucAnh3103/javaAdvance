KẾ HOẠCH HOÀN THIỆN DỰ ÁN (4 NGÀY)
Dựa trên các task còn lại, lộ trình được chia theo từng module liên quan mật thiết với nhau giúp bạn code trôi chảy, tái sử dụng method và dễ debug:

=========================================
 NGÀY 1: QUẢN TRỊ NỀN TẢNG (Admin - Quản lý Danh mục)
Mục tiêu: Hoàn thiện dữ liệu nền (bắt buộc phải có danh mục/hãng sản xuất thì mới thêm được sản phẩm sau đó). Đảm bảo giữ vững cấu trúc 3 lớp (Model-DAO-Service) và PreparedStatement.

[ ] 1.  Hiển thị danh sách hãng sản xuất (danh mục) dạng bảng (2đ)
[ ] 2.  Thêm danh mục: Kiểm tra không được bỏ trống, tên không được trùng (4đ)
[ ] 3.  Sửa danh mục: Kiểm tra ID tồn tại, validate tên sửa không trùng với hãng khác (4đ)
[ ] 4.  Xóa danh mục: Xóa mềm (soft delete) thay vì xóa vật lý bằng cách chuyển is_active/is_deleted (3đ)
*Note: Luôn nhớ bắt exception khi user nhập sai kiểu dữ liệu chữ vào ID (tránh crash).*

=========================================
 NGÀY 2: QUẢN TRỊ SẢN PHẨM (Admin - Quản lý Sản phẩm)
Mục tiêu: CRUD Sản phẩm. Rất nhiều logic validate được tập trung ở đây.

[ ] 1.  Hiển thị danh sách sản phẩm đầy đủ thông tin (kèm tên hãng) và có phân trang (6đ)
[ ] 2.  Thêm mới sản phẩm: Ràng buộc phải nhập ID Hãng, validate bắt buộc giá > 0 và số lượng > 0. Bắt lỗi không bỏ trống (3đ)
[ ] 3.  Sửa sản phẩm: Phải load lại dữ liệu cũ hiển thị lên màn console, không cho phép sửa ID sản phẩm (3đ)
[ ] 4.  Xóa sản phẩm: Phải hỏi xác nhận (Y/N) trước khi xóa thật/xóa mềm (2đ)
[ ] 5.  Tìm kiếm sản phẩm theo tên (tìm kiếm tương đối LIKE '%keyword%') (3đ)
[ ] 6.  Sắp xếp sản phẩm hiển thị theo giá tăng/giảm (2đ)

=========================================
  NGÀY 3: TRẢI NGHIỆM KHÁCH HÀNG & MUA SẮM (Customer - Giỏ hàng & Order)
Mục tiêu: Đứng ở góc độ User để hiển thị hàng hóa, thao tác mua hàng và quản lý đơn của chính họ. Giỏ hàng lưu tạm trong Memory (List) trước khi chốt đơn.

[ ] 1.  Hiển thị sản phẩm còn hàng (stock > 0 & is_active = true) cho người dùng chọn (2đ)
[ ] 2.  Module Giỏ hàng (Tạm): Thêm vào giỏ. Check số lượng yêu cầu <= số lượng tồn kho (3đ)
[ ] 3.  Xác nhận đặt hàng: Viết DAO Insert thông tin vào bảng Orders, lấy Order_ID vừa tạo Insert vào bảng OrderDetails (4đ)
[ ] 4.  Triggers/Logic cập nhật: Tự động dùng lệnh SET stock = stock - ? WHERE id = ? sau khi chốt đơn thành công (3đ)
[ ] 5.  Xem lịch sử đơn hàng của chính User đang đăng nhập và tình trạng đơn (3đ)

=========================================
 NGÀY 4: NGHIỆP VỤ NÂNG CAO & THỐNG KÊ (Admin & Advance)
Mục tiêu: Khép kín hệ thống bằng vòng đời đơn hàng, bổ sung tính năng khó để lấy điểm tối đa. Điểm chú ý: Clean code, review format code.

[ ] 1.  Admin Quản lý Đơn hàng: Hiển thị toàn bộ danh sách đơn hàng của tất cả khách hàng (2đ)
[ ] 2.  Admin Cập nhật đơn hàng: Đổi trạng thái (PENDING → SHIPPING → DELIVERED) (2đ)
[ ] 3.  Nâng cao - Flash Sale: Cài đặt logic giảm giá theo % trong thời gian quy định (thay đổi giá khi hiển thị/chốt đơn) (10đ)
[ ] 4.  Nâng cao - Coupon: Áp dụng mã giảm giá lúc Customer chốt đơn, kiểm tra tính hợp lệ của mã (8đ)
[ ] 5.  Thống kê: Xuất báo cáo (dùng Group By, Order By, Limit) lấy Top 5 sản phẩm bán chạy nhất tháng để Admin xem (7đ)
[ ] 6.  Kiểm tra tổng thể: Review Naming Convention (PascalCase, camelCase), comments, format bảng Console đẹp. (5đ)
